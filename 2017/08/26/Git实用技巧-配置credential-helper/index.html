Git 实用技巧 - 配置 credential.helper

Posted by zihengCat on 2017-08-26

前言

Git的数据交换协议有SSH与HTTP。
使用ssh协议，我们需要在本地配置ssh key，这样一来，每次与服务器代码同步就不需要输入用户名和密码了。
使用http不需要配置，但在每次代码同步时需要我们手动输入用户名和密码，操作比较繁琐。
我们可以使用git credential helper来简化用户名和密码的输入过程。

GitHub 免用户名密码同步

我们以最常用的GitHub为例，配置GitHub免用户名密码代码同步。

创建 Git 存储凭证

我们在用户家目录下创建一个名字为.git-credentials隐藏文件。
将GitHub的用户名与密码按指定格式写入该文件。

$ touch .git-credentials
$ echo -n 'https://<user>:<pass>@github.com' >> ~/.git-credentials

user –> GitHub 用户名
pass –> GitHub 密码。

配置 credential.helper

接下来，我们在.gitconfig配置文件中加入credential.helper选项。

$ git config --global credential.helper store

执行完成后，我们的.gitconfig配置文件中会多出一项[credential]。以后我们使用HTTP与GitHub进行代码同步时，就不需要手动输入用户名与密码了，非常方便！

注意事项

需要注意的是，.git-credentials隐藏文件中的用户名与密码是明文存储的，这有一定的安全隐患。
关于git credential的更多信息，可以查看参考资料。

参考资料
