Linux下搭建Python环境（编译安装）

Compile and Install Python Environment in Linux OS

Posted by zihengCat on 2017-07-06

前言

本文讲解在Linux环境下编译安装Python环境。

在个人使用的CentOS7 Linux系统上，系统默认不安装Python3，yum源中提供的Python3软件包也不是最新版本。

系统自带一枚低版本Python2，为一些系统组件提供支持，不应该随意更动，不然会出现各种各样的Bugs（yum无法正常使用什么的…）。

手动编译安装Python，是Linux系统下搭建Python学习环境的最佳途径。

Step 1 - 编译环境

首先，我们需要在本机安装「基础编译工具链」，这是编译Python源码所必需的。

$ sudo yum install gcc make

代码清单：安装「基础编译工具」

本机安装好gcc，make等基础编译工具，用来编译Python源码（Python语言的官方实现是CPython）。

Step 2 - 获取源码

接下来，前往Python官方网站，下载最新的Python3「源码包」。注意要下载源码包source tarball。

Python官网：https://www.python.org

$ wget https://www.python.org/ftp/python/3.6.1/Python-3.6.1.tar.xz

代码清单：获取Python源码包

解压缩，解包，*.tar.xz压缩包需要分两步解压，*.tar.gz则可以使用tar -xvz -f一步到位。

$ xz -d ./Python-3.6.1.tar.xz
$ tar -xv -f ./Python-3.6.1.tar

代码清单：解压缩Python源码包

亦或者，我们也可以直接把 GitHub 上的cpython镜像仓库克隆到本地。

$ git clone https://github.com/python/cpython.git
$ cd cpython
$ git tag
$ git checkout <tag_version>

代码清单：使用git获取源代码

Step 3 - 安装依赖

cd进入Python源码目录，我们可以先看一下README.rst这个文件，上面告诉了我们编译安装的具体步骤。

在编译安装之前，我们需要先安装一些Python依赖包，这是README上没有提到的（会在./configure检查依赖项时告诉你…）。为了省去后续查找安装依赖的麻烦，这里把所需要的依赖包全部列出。

依赖库 RedHat Debian
zlib zlib-devel zlib1g-dev
bzip2 bzip2-devel libbz2-dev
LZMA xz-devel libxz-dev
UUID uuid-devel libuuid-dev
OpenSSL openssl-devel libssl-dev
Raedline readline-devel libreadline-dev
NCurses ncurses-devel libncursesw5-dev
SQLite sqlite-devel libsqlite3-dev
Tcl/Tk tk-devel tk-dev
gdbm gdbm-devel libgdbm-dev

表：Linux系统下Python依赖库表

# 安装Python依赖库自动化脚本
#/bin/bash
deps="zlib-devel \
      bzip2-devel \
      xz-devel \
      uuid-devel \
      openssl-devel \
      readline-devel \
      ncurses-devel \
      sqlite-devel \
      tcl-devel \
      tk-devel \
      gdbm-devel"

for i in ${deps}
do
    sudo yum -y install ${i}
done

exit ${?}

代码清单: CentOS7 Linux 下安装 Python 依赖库脚本

手写一支shell脚本，把依赖包全部装好。对于Debian系的系统，可以使用apt命令安装相应的依赖包。

Step 4 - 编译安装

依赖包都安装好之后，接下来，我们就按照README的指引，编译安装Python。请确保在Python源代码目录下执行以下操作：

$ ./configure --prefix=/usr/local/python_3.6.1 --enable-optimizations

代码清单：configure编译配置选项

此命令会检查当前系统的编译环境，软件依赖性等等，如果没有问题的话，会成功生成Makefile。

这里我们添加了两个编译选项：

编译选项 解释说明
--prefix 自定义安装路径，默认在/usr/local，个人习惯单独指定一个安装目录，这样管理起来比较方便。
--enable-optimizations 开启编译优化选项，增加编译时间，换取性能提升，这是非常值得的。

表：自定义编译选项表
更多编译选项参看：./configure --help。

$ make
$ sudo make install

代码清单：make编译安装

make编译安装，编译时间取决于机器性能。如果是虚拟机环境，建议把虚拟机性能调高一些。

如果使用--prefix编译选项自定义了安装目录，安装完成后不要忘记了将安装路径加入PATH环境变量中，并将Python3软链接到/usr/bin/下。

$ echo 'export PATH=${PATH}:<path/to/python/bin>' >> ~/.bashrc

代码清单：加入Python到系统路径

Step 5 - 检验安装

命令行下直接启动Python3，如果「安装与部署」都没有问题，会正常进入Python3交互式环境。

$ python3
Python 3.6.1 (default, Jul  6 2017, 15:12:56)
[GCC 4.8.5 20150623 (Red Hat 4.8.5-11)] on linux
Type "help", "copyright", "credits" or "license" for more information.
>>>

代码清单：检验Python安装

至此，最新版Python3已经安装完成，我们可以愉快地使用Python编程了。
