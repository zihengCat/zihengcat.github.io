Linux下搭建Python3环境

Posted by zihengCat on 2017-07-06

前言

本文介绍在Linux环境下编译安装Python3。

编译安装Python3

在子恒喵使用的CentOS7 Linux 系统中，默认的yum源是没有最新版Python3的安装包的。
系统自带的是一枚老版本的Python2，而且精简过，只为一些系统应用提供支持，我们不要随意更动，否则会出现各种各样的问题…(比如yum不能用)
所以，手动编译安装Python3，是Linux下搭建Python学习环境的最佳选择。

Step 1 编译环境

我们需要安装几个基础的编译工具包，这是编译源码所必需的。

$ sudo yum install gcc make
$ sudo yum install wget
  • gcc，make用来编译Python源码（Python语言的官方实现是CPython）
  • wget是Linux下一个好用的下载工具，可以用来下载Python源码包（大部分Linux系统都会默认安装）

Step 2 获取源码

接下来，我们前往Python官方网站下载最新的Python3源码包。
注意，一定要下载源码包source tarball哦。

$ wget https://www.python.org/ftp/python/3.6.1/Python-3.6.1.tar.xz

解压缩，解包（xz压缩包要分两步解压，gz就可以在tar命令中一步搞定）

$ xz -d ./Python-3.6.1.tar.xz
$ tar -xv -f ./Python-3.6.1.tar

嗯，80+MB的源码，XZ直接给压到16MB…相当强大。

Step 3 安装依赖

cd进入Python3源码目录，我们可以先看一下README.rst这个文件，上面告诉了我们编译安装的具体步骤。
但是，在编译安装之前，我们需要先安装一些Python依赖包，这是README上没有提到的。(会在configure检查依赖时告诉你…)
为了省去后续查找安装依赖的麻烦，子恒喵在这里把所需要的依赖包都写出来。

#/bin/bash

deps="zlib-devel \
      bzip2-devel \
      openssl-devel \
      readline-devel \
      ncurses-devel \
      sqlite-devel \
      tcl-devel \
      tk-devel \
      gdbm-devel"

for i in ${deps}
do
    sudo yum -y install ${i}
done

exit ${?}

手写一段shell脚本，把依赖包都装上。
debian系的系统，可以在远端仓库里寻找相应的依赖包。

Step 4 编译安装

依赖都安装好之后，接下来，我们就按照README的指引，编译我们的Python3吧。
请确保在Python源代码目录下执行以下操作:

$ ./configure --prefix=/usr/local/python-3.6.1 --enable-optimizations

此命令会检查当前系统的编译环境，软件依赖性等等，如果没有问题的话，会成功生成Makefile。
这里我们添加了两个参数:

  • --prefix 自定义安装路径，默认在/usr/local，子恒喵习惯单独指定一个安装目录，这样管理起来会比较方便
  • --enable-optimizations 开启编译优化选项，增加编译时间，换取性能提升，这是值得的。

更多编译参数可以查看: ./configure --help

$ make
$ sudo make install

编译安装，编译时间取决于机器性能。如果是虚拟机环境，建议把虚拟机性能调高一些。
如果自定义了安装路径，别忘了将路径加入PATH环境变量中，并将python3链接到/usr/bin/哦。^_^

$ echo 'export PATH=${PATH}:<prefix/bin>' >> ~/.bashrc

Step 5 检验安装

$ python3
Python 3.6.1 (default, Jul  6 2017, 15:12:56)
[GCC 4.8.5 20150623 (Red Hat 4.8.5-11)] on linux
Type "help", "copyright", "credits" or "license" for more information.
>>>

至此，最新版Python3已经安装完成，愉快地使用Python编程吧!
